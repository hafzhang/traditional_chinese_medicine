# 通用任务拆分指南

## 何时需要拆分任务

### 拆分信号
- 预计时间 > 1 小时
- 涉及 > 3 个文件
- 包含 > 2 个独立步骤
- 需要 > 1 次代码审查

## 通用拆分策略

### 按开发阶段拆分
1. 数据结构/模型定义
2. 接口/函数签名
3. 核心逻辑实现
4. 错误处理
5. 测试验证

### 按功能层次拆分
1. 基础框架
2. 核心功能
3. 辅助功能
4. 优化改进

### 按文件/模块拆分
1. 数据模型文件
2. 业务逻辑文件
3. 接口文件
4. 工具函数文件

## 通用子任务模板

### 1. 创建文件
- [ ] 创建新文件
- [ ] 添加必要的导入
- [ ] 定义基础结构

### 2. 实现核心功能
- [ ] 实现主要逻辑
- [ ] 添加参数验证
- [ ] 处理边界情况

### 3. 集成
- [ ] 与现有代码集成
- [ ] 更新相关接口
- [ ] 处理兼容性

### 4. 测试
- [ ] 编写单元测试
- [ ] 手动测试
- [ ] 修复 bug
