## 📖 文档说明

### 00_design_system.md - 设计系统总览

设计系统的核心规范文档，包含：

- **设计原则**：科学性、易用性、一致性、可访问性
- **色彩系统**：主色调、体质颜色（9种）、功能色彩、渐变色系统
- **排版系统**：字体大小、字重、行高
- **间距系统**：从小到大的间距规范
- **圆角系统**：不同级别的圆角规范
- **阴影系统**：三级阴影规范
- **组件规范**：卡片、按钮、标签、图标
- **响应式设计**：断点系统和适配规则
- **动画系统**：过渡时间和缓动函数
- **可访问性标准**：对比度和点击区域
- **页面导航流程**：完整的用户流程图

### 01_index_page.md - 首页设计文档

首页 (`/pages/index/index`) 的完整设计规范：

- **页面结构**：Hero区域、测试说明、九种体质、功能特点、科学依据
- **组件设计**：Hero区域、体质网格（3x3）、功能特点列表
- **数据流**：从 `CONSTITUTION_INFO` 获取数据
- **用户交互**：开始测试、查看体质详情

### 02_test_page.md - 测试页面设计文档

测试页面 (`/pages/test/test`) 的完整设计规范：

- **页面结构**：进度条、问题卡片、快速跳转面板
- **题目分组**：按9种体质分组的题目范围
- **组件设计**：进度条、体质标签、选项列表、快速跳转网格
- **状态管理**：题目数据、答案数组、当前索引
- **API 调用**：获取题目、提交答案
- **用户交互流程**：完整的测试流程

### 03_result_page.md - 结果页面设计文档

结果页面 (`/pages/result/result`) 的完整设计规范：

- **页面结构**：动态颜色头部、体质特征、分数图表、次要体质、调理建议
- **组件设计**：结果头部、特征预览、分数图表、次要体质列表
- **数据流**：从 API 获取结果，从本地获取体质详情
- **分数可视化**：按分数排序的进度条图表
- **用户交互**：查看详情、饮食推荐、重新测试

### 04_detail_page.md - 详情页面设计文档

详情页面 (`/pages/detail/detail`) 的完整设计规范：

- **页面结构**：体质头部、特征、调理原则、禁忌事项、科学依据
- **组件设计**：体质头部、特征列表、调理建议、禁忌网格
- **数据完整性**：所有体质必须包含的数据结构
- **条件渲染**：心理特征、情志调节的可选显示
- **用户交互**：查看饮食推荐、返回结果页

### 05_food_page.md - 饮食推荐页面设计文档

饮食推荐页面 (`/pages/food/food`) 的完整设计规范：

- **页面结构**：饮食头部、宜吃食物、不宜食物、推荐食谱、饮食原则
- **组件设计**：食物卡片、标签系统、食谱列表、原则列表
- **食物标签系统**：性质标签（温热平凉寒）、味道标签（酸甜苦辛咸）
- **API 调用**：获取饮食推荐数据

## 🔧 使用指南

### 前端开发者

1. **开始新功能开发前**
   - 阅读相关页面的设计文档
   - 了解设计系统中的组件规范
   - 确认数据流和 API 调用方式

2. **开发过程中**
   - 参考组件规范进行样式实现
   - 遵循色彩系统和排版系统
   - 确保可访问性标准

3. **代码审查前**
   - 检查是否符合设计规范
   - 确认响应式适配
   - 验证可访问性

### UI/UX 设计师

1. **设计新页面时**
   - 参考设计系统的规范
   - 保持与现有页面的一致性
   - 更新相关文档

2. **修改现有设计时**
   - 同步更新设计文档
   - 标注版本变更
   - 通知开发团队

### 产品经理

1. **规划新功能时**
   - 参考页面设计文档
   - 了解现有交互模式
   - 评估设计复用性

2. **编写需求文档时**
   - 引用相关设计规范
   - 确保技术可行性
