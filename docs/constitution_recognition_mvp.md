# 中医体质识别 — MVP 开发方案

---

## 一、目标（Goal）

在最短周期内交付可上线验证的最小可行产品（MVP），以验证用户对"快速体质识别 + 个性化饮食/作息建议"的需求与转化路径。

### 1.1 核心验证目标

| 验证维度 | 指标 | 目标值 |
|---------|-----|--------|
| 用户完成率 | 从进入到完成测试的比例 | ≥ 60% |
| 测试时长 | 平均完成时间 | ≤ 8 分钟 |
| 报告分享率 | 完成后分享的比例 | ≥ 10% |
| 推荐点击率 | 查看详细方案的比例 | ≥ 15% |
| 判定准确率 | 与60题标准版对比 | ≥ 80% |

### 2.3 问卷题目分配（30题精简版）

基于王琦院士 CCMQ 标准量表，精简至30题：

| 体质类型 | 题目数量 | 覆盖内容 | 题目编号 |
|---------|---------|---------|---------|
| 平和质 | 4题 | 精力充沛、睡眠良好、胃口正常、适应力强 | Q1-Q4 |
| 气虚质 | 4题 | 疲乏气短、容易自汗、容易感冒、肌肉松软 | Q5-Q8 |
| 阳虚质 | 4题 | 手足不温、喜热饮食、畏寒怕冷、精神不振 | Q9-Q12 |
| 阴虚质 | 4题 | 口干咽燥、盗汗、手足心热、易心烦 | Q13-Q16 |
| 痰湿质 | 3题 | 体形肥胖、身重不爽、面部油腻 | Q17-Q19 |
| 湿热质 | 3题 | 面垢油光、口苦口臭、大便黏滞 | Q20-Q22 |
| 血瘀质 | 3题 | 面色晦暗、易有瘀斑、疼痛部位固定 | Q23-Q25 |
| 气郁质 | 3题 | 情绪抑郁、胸胁胀满、常爱叹气 | Q26-Q28 |
| 特禀质 | 2题 | 容易过敏、喷嚏连连、哮喘 | Q29-Q30 |

**评分标准**：5分制 Likert 量表
- ①没有（1分）  ②很少（2分）  ③有时（3分）  ④经常（4分）  ⑤总是（5分）

## 四、核心功能拆解与验收标准

### 4.1 问卷答题流程

**功能描述**：题目逐步展示，支持进度提示与答题提交

**验收标准**：
- 完成一套30题问卷，从开始到提交不超过8分钟
- 支持进度保存（本地缓存）
- 提交后返回200并含答卷ID

### 4.2 判定服务（后端）

**功能描述**：接收答卷，按题目映射到9种体质维度，计算分数并按既定阈值判定主要/次要体质

**判定规则**：
```
第一步：计算各维度原始分
  每个体质维度包含若干相关问题，按题目归属累加得分

第二步：转换为百分制
  转换分数 = 原始分 × 2.5  （100分制）

第三步：判定体质类型
  • 平和质判定：平和质分数 ≥ 60分，且其他8种体质分数 < 40分
  • 偏颇体质判定：某体质分数 ≥ 40分 → 主要体质
  • 兼夹体质判定：多个体质分数 ≥ 30分 → 次要体质
```

**验收标准**：
- 判定结果字段包含primary、secondary、scores
- 单次平均响应时间 < 300ms（无冷启动）
- 判定准确率 ≥ 80%

**响应数据结构**：
```json
{
  "result_id": "uuid",
  "primary_constitution": "qi_deficiency",
  "primary_constitution_name": "气虚质",
  "secondary_constitutions": [
    {"type": "yang_deficiency", "name": "阳虚质", "score": 35}
  ],
  "scores": {
    "peace": 25,
    "qi_deficiency": 85,
    "yang_deficiency": 35,
    "yin_deficiency": 20,
    "phlegm_damp": 15,
    "damp_heat": 10,
    "blood_stasis": 5,
    "qi_depression": 25,
    "special": 10
  },
  "created_at": "2025-01-13T10:30:00Z"
}
```

### 4.3 报告生成与可视化

**功能描述**：生成可阅读的报告，包含体质标签、特征、调理要点、饮食建议；展示雷达图

**报告内容结构**：
```

│    您的体质：气虚质（兼夹：阳虚质）                          │
│                                                              │
│  【体质特征】                                                │
│  气虚质的典型表现：                                           │
│  ✓ 疲乏气短，活动后加重                                      │
│  ✓ 容易出汗，稍动即汗                                        │

│                                                              │

│                                                              │
│  【调理要点】                                                │
│  气虚质调理原则：补气健脾                                    │

│  • 运动：选择柔和运动，避免大汗淋漓                          │
```

**验收标准**：
- 报告页面可在移动端和小程序里正确渲染
- 关键内容可在报告中一屏可见
- 雷达图交互流畅

### 4.4 饮食推荐（基础）

**功能描述**：从静态食物库按体质推荐宜/忌食物与菜谱示例

**推荐内容**：
| 推荐类别 | 内容 | 示例（气虚质） |
|---------|-----|---------------|
| **宜吃食物** | 补气健脾食物 | 糯米、大枣、山药、鸡肉、牛肉、香菇 |
| **忌吃食物** | 耗气破气食物 | 山楂、大蒜、薄荷、荷叶、绿豆 |
| **推荐食谱** | 体质食疗方 | 黄芪炖鸡、山药粥、红枣茶 |
| **饮食原则** | 饮食建议 | 少食多餐、细嚼慢咽、忌生冷 |

**食物库数据结构**：
```json
{
  "food_id": "F001",
  "name": "山药",
  "nature": "平",
  "flavor": "甘",
  "meridians": ["脾", "肺", "肾"],
  "suitable_constitutions": ["qi_deficiency", "yin_deficiency"],
  "avoid_constitutions": ["phlegm_damp"],
  "nutrition_info": {
    "calories": 56,
    "protein": 1.9,
    "carbs": 11.6,
    "fat": 0.2

  "recipes": ["山药粥", "蒸山药", "山药炖排骨"]
}
```

**验收标准**：
- 每种体质至少返回5条宜吃/3条忌吃和1条菜谱
- 推荐结果可通过API获取

### 4.5 结果存储与复测入口

**功能描述**：将体质结果与用户ID（或匿名ID）关联，支持复测查看历史

**验收标准**：
- 数据库能查询到最近一次结果
- 复测会创建新记录并可对比历史分数

### 4.6 分享卡片与小程序码（简版）

**功能描述**：生成一张可分享的体质卡片并支持生成小程序码或分享图片

**验收标准**：
- 可下载/分享图片或复制小程序链接

### 6.2 数据库设计

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    douyin_openid VARCHAR(128) UNIQUE,
    wechat_openid VARCHAR(128) UNIQUE,
    anonymous_id VARCHAR(128) UNIQUE,
    nickname VARCHAR(100),
    avatar_url VARCHAR(500),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 问卷题目表
CREATE TABLE questions (
    id SERIAL PRIMARY KEY,
    question_number INT NOT NULL,  -- 1-30
    content TEXT NOT NULL,  -- 题目内容
    constitution_type VARCHAR(50) NOT NULL,  -- 对应体质类型
    options JSONB,  -- 选项配置
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 体质结果表
CREATE TABLE constitution_results (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id),
    primary_constitution VARCHAR(50) NOT NULL,
    secondary_constitutions JSONB,
    scores JSONB NOT NULL,  -- 九种体质分数
    answers JSONB,  -- 用户答案
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 食物库表
CREATE TABLE foods (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    nature VARCHAR(20), -- 寒、凉、平、温、热
    flavor VARCHAR(50), -- 酸、苦、甘、辛、咸
    meridians TEXT[], -- 归经
    suitable_constitutions TEXT[],
    avoid_constitutions TEXT[],
    nutrition_info JSONB,
    effects TEXT[],
    recipes TEXT[],
    image_url VARCHAR(500)
);

-- 菜谱表
CREATE TABLE recipes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(200) NOT NULL,
    suitable_constitutions TEXT[],
    ingredients JSONB,
    steps JSONB,
    images TEXT[],
    description TEXT
);
```

 ## 九、风险与缓解措施

### 9.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|-----|-----|---------|
| 问卷精简导致判定准确率下降 | 用户信任度下降 | 中 | 上线后A/B测试30题/60题对比，动态调整题目权重 |
| 小程序审核不通过 | 上线延期 | 低 | 提前准备合规材料，预留Web/H5备用入口 |
| 高并发下性能问题 | 用户体验差 | 低 | 使用缓存、CDN、Serverless自动扩容 |

### 9.2 合规风险

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|-----|-----|---------|
| 医疗用语违规 | 小程序下架 | 中 | 文案使用"养生/建议"措辞，避免"治疗/诊断"用语 |
| 用户隐私泄露 | 法律风险 | 低 | 数据加密存储，符合个人信息保护法 |
| 虚假宣传风险 | 监管处罚 | 低 | 加入免责声明，不承诺治疗效果 |

### 9.3 产品风险

| 风险 | 影响 | 概率 | 缓解措施 |
|-----|-----|-----|---------|
| 用户完成率低 | 数据不足 | 中 | 优化问卷流程，增加进度提示 |
| 推荐内容质量差 | 用户投诉 | 中 | 建立内容审核机制，专家复核 |
| 分享传播率低 | 增长受限 | 中 | 设计有吸引力的分享卡片，增加社交话题性 |
